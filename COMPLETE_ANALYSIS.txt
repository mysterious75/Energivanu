PROBLEM 1 — OVERFITTING (MOST CRITICAL)
----------------------------------------
  Symptoms:
    - Val loss increases from epoch 1
    - Best MAE at epoch 1 (9.90 MW with sync spikes)
    - Model memorizes noise instead of patterns
  Root causes:
    - Synthetic data too predictable (smooth patterns)
    - Random sync spikes are UNPREDICTABLE (no model can learn random)
    - 414K-518K samples insufficient for large models (6.5M params)
    - Large batch sizes (256-512) → sharp minima → poor generalization
  Current mitigations:
    - dropout=0.35, weight_decay=3e-4
    - batch=128 (compromise)
    - Removed sync spikes from data

PROBLEM 2 — DIRECTION ACCURACY AT RANDOM (50%)
------------------------------------------------
  Symptoms:
    - DirAcc never exceeds 50% in any experiment

    - Loss function (MSE) doesn't penalize wrong direction specifically
  Impact: Battery needs direction signal for charge/discharge decisions

5. EXISTING SOLUTIONS FROM RESEARCH/WEB
================================================================================

SOLUTION A — PI-DLinear (arXiv 2605.04074, Apr 2026)
------------------------------------------------------
  WHAT: Physics-Informed DLinear for GPU power forecasting
  WHO: Mohammad AlShaikh Saleh, Sanjay Chawla et al.
  LINK: https://arxiv.org/abs/2605.04074
  HOW IT WORKS:
    - Physics-aware model using thermal RC network (Newton's cooling law)
    - ODE-based modeling linking GPU compute/memory utilization + temperature
    - 0.78%-39% better MSE than SOTA transformers
    - 0.99%-51.82% better MAE than SOTA transformers
    - Tested on REAL data center data
    - Handles power throttling and load transients
  RELEVANCE: HIGH — Directly addresses GPU power forecasting on real data
  CAN WE USE IT? YES — DLinear is simpler than Transformer, less overfitting

SOLUTION B — INFORMER (AAAI 2021 Best Paper)
-----------------------------------------------
  WHAT: ProbSparse self-attention for long sequence time-series
  WHO: Zhou Haoyi et al.
  LINK: https://github.com/zhouhaoyi/Informer2020
  HOW IT WORKS:
    - O(L log L) attention complexity instead of O(L²)
    - ProbSparse attention selects "active" queries only
    - Generative decoder for single-step long sequence output
    - Self-attention distilling reduces memory
  RELEVANCE: MEDIUM — Already mentioned in 05_NEXT_STEPS.md
  CAN WE USE IT? YES — But complex to implement

SOLUTION C — AUTOFORMER (NeurIPS 2021)
-----------------------------------------
  WHAT: Decomposition architecture with auto-correlation
  WHO: Wu Haixu et al.
  LINK: https://github.com/thuml/Autoformer
  HOW IT WORKS:
    - Series decomposition (trend + seasonal) as inner operation
    - Auto-correlation mechanism replaces self-attention
    - Progressive decomposition across blocks
  RELEVANCE: MEDIUM — Already mentioned in 05_NEXT_STEPS.md
  CAN WE USE IT? YES — But may not help with fundamental overfitting

SOLUTION D — PATCHTST (ICLR 2023)
-----------------------------------
  WHAT: Patch Time Series Transformer (channel-independent)
  WHO: Nie et al.
  HOW IT WORKS:
    - Patching time series into sub-series
    - Channel-independent (each univariate series processed separately)
    - Better than multivariate Transformers on many benchmarks
  RELEVANCE: HIGH — Patching concept already in code (patch_size=10)
  CAN WE USE IT? YES — Can modify current code to use PatchTST style

SOLUTION E — EnergAIzer (IBM, ISPASS 2026)
--------------------------------------------
  WHAT: Fast GPU power estimation framework
  WHO: IBM Research (Kyungmi Lee et al.)
  HOW IT WORKS:
    - Lightweight performance model predicts utilization inputs
    - 8% power error on NVIDIA Ampere, 7.6% on H100
    - Frequency scaling + architectural config exploration
  RELEVANCE: LOW — More for hardware design than forecasting

SOLUTION F — LINIER TRANSFORMERS ARE BETTER FOR TSF
-----------------------------------------------------
  WHAT: Research showing simple linear models > complex Transformers for TSF
  WHO: Zeng et al. (2023) "Are Transformers Effective for Time Series Forecasting?"
  KEY FINDING:
    - Simple linear models beat Transformer on many benchmarks
    - Multi-headed self-attention causes permutation invariance (bad for time series)
    - Order matters in time series but attention is permutation-invariant
  RELEVANCE: HIGH — Explains why simpler model may work better
  CAN WE USE IT? YES — Try DLinear or NLinear as baseline

SOLUTION G — TSMixer (Google/2023)
------------------------------------
  WHAT: All-MLP architecture for time series
  KEY FINDING:
    - MLP-based models outperform Transformers on many TSF benchmarks
    - Much fewer parameters, less overfitting
  RELEVANCE: MEDIUM — Worth trying as simpler alternative

SOLUTION H — ZERO-SHOT FORECASTING (Chronos/Amazon)
